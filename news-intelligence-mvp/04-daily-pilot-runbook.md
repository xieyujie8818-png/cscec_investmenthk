# 04 每日人工半自動試運行手冊



本手冊用於MVP前4週，在飛書機器人完全接入前，先用人工半自動方式跑通內容質量和推送節奏。



當前MVP採用固定範圍和固定來源：**日報系統（daily-brief-app）自動採集四報官網列表主欄**；人工查漏時可使用地產站或站內搜索（見 `02-source-list.md`）。**不再使用** HKET 電子報 flipbook（A0/D0）。每日對外呈現以 **《香港簡訊》** 為準（目錄錨點+摘要+原文全文），服務決策層與部門同事；暫不開放同事自行修改摘取範圍或新增來源。



## 每日角色



| 角色 | 建議人數 | 職責 |

| --- | --- | --- |

| 當日信息整理人 | 1人 | 掃描四報列表主欄、初篩、填表、組裝《香港簡訊》 |

| 當日業務審核人 | 1人 | 確認選題、口徑、評分與去重 |

| 推送負責人 | 1人，可兼任 | 發飛書群簡訊；另發市場部同事月報標記 |

| 市場部日報/月報負責人 | 1人 | 接收月報素材標記，沉澱月報素材池 |



## 每日時間表（參考6月2日試點）



| 時間 | 動作 | 產出 |

| --- | --- | --- |

| 09:00-09:15 | 系統/人工：掃描四報列表主欄（標題預篩 → 匹配者抓正文） | 候選新聞池 |

| 09:15-09:35 | 選題、去重、填寫新聞登記表 | 約15-20條定稿候選 |

| 09:35-10:10 | 組裝《香港簡訊》：目錄錨點 + 每條原文全文 | `daily-leadership-brief.md` |

| 10:10-10:15 | 業務審核 | 定稿 |

| 10:15-10:20 | 整理飛書群簡訊（無月報標記、無原文鏈接） | `daily-feishu-digest.md` |

| 10:20前 | 發送飛書群；發市場部負責人內部標記 | 完成每日推送 |

| 下班前 | 市場部負責人更新月報素材池 | `daily-internal-monthly-markers.md` + 登記表 |



**多日補編：** 若一次編制多個日期，候選去重後匯總為 **同一份** 定稿，見 `08-daily-agent-workflow.md`。



## 操作步驟



### 1. 收集候選新聞



從四報**官網列表主欄**開始閱讀（排除側邊欄、評論欄）；先以標題匹配專題關鍵詞，**僅通過預篩者**才點入 `article` 取正文。**不要使用 flipbook、全站搜索、地產站。**



候選新聞納入條件：



- 以 **當日或選定區間** 更新為主；重大政策、成交或企業財政可回溯至30天內且仍具研判價值。

- 涉及 `01-scope-taxonomy.md` 中固定專題或日報欄目。

- **金融欄**：宏觀經濟、香港政策為主，排除個股短訊。

- **只要新聞稿**，排除評論、專欄、樓評。

- 會員文章由具備權限同事摘取。



### 2. 選題與欄目



對外《香港簡訊》欄目：



| 欄目 | 建議 | 內容 |

| --- | --- | --- |

| 【金融與經濟】 | 總量內靈活分配 | 宏觀、政策、PMI、經貿、產業政策等 |

| 【建築與地產】 | 總量內靈活分配 | 新盤、成交、租金、學生宿舍、改建等 |

| 【北都專題】 | 總量內靈活分配 | 北都產業、大學城、土地、洪水橋等 |



**定稿總量 15–20 條**；去重；評分 ≥70。不再硬性要求各欄固定條數。



### 3. 組裝《香港簡訊》正文



使用 `templates/daily-leadership-brief.md`：



1. 先寫 **目錄**：每條標題為錨點鏈接，下方附一至兩句摘要（**直接取自原文首句**，方括號【】包裹）；**不含**原文鏈接。

2. 再寫 **正文**：每條含 `[來源：報社名]`、`【本報M月D日消息】`（用該文發佈日期）、**原文全文**（照錄，不做 AI 改寫）。

3. **對外定稿不含原文鏈接**；內部登記表可保留 `source_url` 供查證。



### 4. 填寫新聞登記表



使用 `templates/news-register.csv` 或飛書多維表格。必填：標題、日期、來源、鏈接（內部）、日報欄目、摘要、評分、人工狀態。



### 5. 評分與排序



按 `03-scoring-rules.md` 打分，供內部選題參考；**評分不寫入飛書群推送或領導版正文**。



### 6. 雙通道推送



| 通道 | 模板 | 對象 | 內容 |

| --- | --- | --- | --- |

| 對外每日推送 | `daily-feishu-digest.md` | 部門飛書群（眾人） | 今日提要 + 三欄目標題/原文首句摘要 + 完整版鏈接（**無原文鏈接**） |

| 內部月報標記 | `daily-internal-monthly-markers.md` | 市場部負責日報/月報同事 | 月報候選、跟進事項、未推送歸檔 |



**禁止** 在眾人推送中包含「月報素材標記」「內部評分」「推送對象建議」等欄位。



### 7. 飛書與自動化



MVP階段人工發送；格式穩定後按 `05-feishu-it-requirements.md` 對接機器人。機器人亦應區分 `audience_type`: `department_digest`（群推送）與 `monthly_marker`（指定人）。



## 每週復盤



每週五15分鐘：



- 領導或同事反饋哪些條目有用。

- 四報哪些版面/檢索詞最有效。

- 15–20 條總量是否合理。

- 月報素材池是否足夠（由市場部負責人匯報）。
